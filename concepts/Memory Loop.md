# Memory Loop

Memory is a loop, not a pile of notes.

Claude reads the workspace, does the work, writes a trace, and then you promote only the lesson worth keeping.

## The loop

1. Start from [[The Script]].
2. Work from real files and examples.
3. Save the actual work in the workbench.
4. Promote the durable lesson into the right memory folder.
5. Start the next session with better context.
