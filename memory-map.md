# Memory Map

Open this file in Obsidian when you want to see how the workspace fits together.

This is not a task list. It is the small map that helps Claude and you find the right context faster.

## Core loop

- [[The Script]] tells Claude what to read first.
- [[Memory Loop]] explains how a session turns into reusable context.
- [[Workbench vs Memory]] keeps finished work separate from lessons worth keeping.
- [[Promotion]] is the end-of-session move that makes the next session better.
- [[Skills]] shows the slash commands you can use. The real instructions live in `.claude/skills/`, which Obsidian hides.

## Daily use

1. Start in `CLAUDE.md`.
2. Do the work in `copy/` or `creatives/`.
3. Save context in `daily/`.
4. Promote durable lessons into `winners/`, `losers/`, `brand/`, `clients/`, or `workflows/`.
5. Use `skills/Skills.md` when you want to see what slash commands are available.
6. Add a short `Related:` line to durable notes so Obsidian backlinks can show the pattern.

## Link rule

Use links when they explain why a note matters.

Good links:

- a client note linking to a workflow it uses
- a winner linking to the swipe or mechanism behind it
- a daily note linking to the client and workflow touched that day
- a new rule linking back to [[Promotion]] or [[Workbench vs Memory]]

Skip links that are only decoration. Three useful links beat twenty noisy ones.
