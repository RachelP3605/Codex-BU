# advanced-script-interview

Command: `/advanced-script-interview`

Use this after Obsidian works.

It runs a deeper interview about how you work, what Claude should remember, where detail should live, what should stay out of the Script, and which links should become part of the workspace map.

It does not edit files right away. First it reads the workspace, interviews you one question at a time, and shows a proposed change plan.

Actual runtime file: `.claude/skills/advanced-script-interview/SKILL.md`
