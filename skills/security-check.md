# security-check

Command: `/security-check`

Use this before installs, GitHub backup, public sharing, client work, bypass permissions, or risky file edits.

It checks for secrets, public/private mistakes, dangerous commands, dependency risk, rollback state, and whether the action is limited to the intended folder.

It ends with one of three results: approve, fix first, or stop.
